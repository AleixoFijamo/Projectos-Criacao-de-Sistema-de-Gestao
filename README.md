# Projecto 01: Gestão de Agentes

  # 1.1. Resumo do Projeto
Este projeto apresenta um sistema web robusto para a gestão eficiente de agentes e controlo de equipamentos, desenvolvido para otimizar as operações de campo e a administração de recursos.

  # 1.2. Problema
No contexto de operações comerciais com uma vasta rede de agentes e equipamentos, a gestão manual ou fragmentada pode levar a ineficiências significativas, erros de dados e dificuldades na monitorização do desempenho. Este sistema foi concebido para centralizar e automatizar o controlo de agentes e equipamentos.
Os principais desafios abordados incluem: 
* Rastreamento de Equipamentos: que se refere à dificuldade em saber qual equipamento está atribuído a qual agente e o seu estado (ativo, inativo, manutenção)
* Gestão de Agentes: que envolve o acompanhamento do perfil, localização, supervisor e desempenho de cada agente

# Projecto 02: Rastreiamento de Encomendas

  # 2.1. Resumo do Projecto
É um sistema web intuitivo e eficiente para o rastreamento de encomendas, focado na rota Portugal → Moçambique. Desenvolvido para proporcionar transparência e atualizações em tempo real, o sistema permite que clientes e administradores monitorem o progresso das entregas desde a recolha até à entrega final.

  # 2.2. Problema
Este sistema soluciona a falta de informação em tempo real sobre o estado das remessas, minimizando a ansiedade dos clientes e otimizando a gestão logística para a empresa. Garante que tanto clientes quanto a equipa administrativa tenham acesso fácil e rápido a informações cruciais sobre cada encomenda.

# Projecto 03: Gestão de Delivery

  # 3.1. Resumo do Projecto
É um sistema web de gestão logística desenvolvido para pequenas e médias empresas de entregas (delivery/motoboy), permitindo controlar todo o ciclo de vida de um pedido — desde o registo até à entrega final. O sistema conta com autenticação de utilizadores, dashboard com indicadores em tempo real, gestão de pedidos, gestão de equipa de entregadores ("Central MotoBoy"), geração de relatórios exportáveis e administração de utilizadores com diferentes níveis de acesso.

  # 3.2. Problema
Empresas de entregas de pequena e média dimensão frequentemente gerem os seus pedidos de forma manual — através de cadernos, folhas de cálculo ou grupos de WhatsApp — o que gera perda de informação, dificuldade em acompanhar o estado das entregas em tempo real, ausência de histórico organizado e falta de controlo sobre os motoboys disponíveis.
O Fly Way Delivery resolve estes problemas ao centralizar:
* Registo: registar pedidos com dados do cliente, tipo de entrega, localização e valor da taxa;
* Acompanhamento: acompanhar o estado de cada pedido (Pendente, Em Rota, Entregue, Cancelado) de forma visual e organizada;
* Motoboy: Atribuir entregas a motoboys específicos e gerir a disponibilidade da equipa;
* Relatório: Exportar relatórios filtrados (por data, cliente, status, tipo ou motoboy) em CSV e Excel, facilitando a análise do negócio;
